
## **Barcodes (1D)**

### **Structure**
- Series of **parallel black and white bars** of varying widths
- Each pattern represents numbers/characters
- Typically stores 20-25 characters


## **QR Codes (2D)**

### **Structure**

### **Key Components**
- **Position markers**: Three corner squares for orientation
- **Alignment patterns**: For large codes, correction of distortion
- **Timing patterns**: Determine matrix size
- **Data area**: Actual encoded information
- **Error correction**: Reed-Solomon codes (up to 30% recovery)

### **How It Works**
1. **Encoding**: Data converted to binary, divided into blocks
2. **Error correction added**: Allows damage(30%) recovery
3. **Matrix creation**: Binary placed in 2D grid pattern
4. **Scanning**: Camera captures entire image
5. **Detection**: Software finds position markers
6. **Correction**: Adjusts for angle/distortion
7. **Decoding**: Reads data modules, applies error correction

### **Capacity**
- **Numeric**: Up to 7,089 characters
- **Alphanumeric**: Up to 4,296 characters
- **Binary**: Up to 2,953 bytes


**QR** = **Quick Response**
